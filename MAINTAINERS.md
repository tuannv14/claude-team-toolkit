## How to become a maintainer

Maintainers are added based on sustained contribution. The path:

1. Open and merge several quality PRs (bug fixes, new skills following the pattern, documentation improvements).
2. Help triage incoming issues — reproduce, label, suggest fixes.
3. Demonstrate alignment with the security model and token-efficiency principles.
4. Existing maintainers invite you.

Maintainers commit to:
- Reviewing PRs within ~7 days
- Responding to security reports within 72 hours (see [SECURITY.md](SECURITY.md))
- Following the [Code of Conduct](CODE_OF_CONDUCT.md)
